public V put(K key, V value) {
    return putVal(key, value, false);
}


final V putVal(K key, V value, boolean onlyIfAbsent) {
    if (key == null || value == null) throw new NullPointerException();
    int hash = spread(key.hashCode());
    int binCount = 0;
    for (Node<K,V>[] tab = table;;) {
        Node<K,V> f; int n, i, fh;
        if (tab == null || (n = tab.length) == 0)
            tab = initTable();
        else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
            if (casTabAt(tab, i, null,
                         new Node<K,V>(hash, key, value, null)))
                break;                   // no lock when adding to empty bin
        }
        else if ((fh = f.hash) == MOVED)
            tab = helpTransfer(tab, f);
        else {
            V oldVal = null;
            synchronized (f) {
                if (tabAt(tab, i) == f) {
                    if (fh >= 0) {
                        binCount = 1;
                        for (Node<K,V> e = f;; ++binCount) {
                            K ek;
                            if (e.hash == hash &&
                                ((ek = e.key) == key ||
                                 (ek != null && key.equals(ek)))) {
                                oldVal = e.val;
                                if (!onlyIfAbsent)
                                    e.val = value;
                                break;
                            }
                            Node<K,V> pred = e;
                            if ((e = e.next) == null) {
                                pred.next = new Node<K,V>(hash, key,
                                                          value, null);
                                break;
                            }
                        }
                    }
                    else if (f instanceof TreeBin) {
                        Node<K,V> p;
                        binCount = 2;
                        if ((p = ((TreeBin<K,V>)f).putTreeVal(hash, key,
                                                       value)) != null) {
                            oldVal = p.val;
                            if (!onlyIfAbsent)
                                p.val = value;
                        }
                    }
                }
            }
            if (binCount != 0) {
                if (binCount >= TREEIFY_THRESHOLD)
                    treeifyBin(tab, i);
                if (oldVal != null)
                    return oldVal;
                break;
            }
        }
    }
    addCount(1L, binCount);
    return null;
}

从构造方法和put方法大致可以看出，JDK8中的数据结构与JDK7中的ConcurrentHashMap有了较大的区别，JDK8中已经没有Segment数组了。在JDK8中，只有一个数组，与HashMap的数据结构一致了。

在putVal方法中，先计算hash值，然后进入循环。一开始table为null，所以tab也为null，需要对其进行初始化，即initTable方法。该方法没有特别需要注意的，这里略过。

因为是一个for循环，而第一个if分支中没有break，所以第二次进入循环时tab不为null，会进入到下面的分支。第二个分支就是计算出索引位，如果该位置上为null，则实例化一个Node结点，用原子操作将其放入到该位置。

接下来看第三个分支，即hash值为MOVED的情况。MOVED是一个常量，表示该ConcurrentHashMap正在进行扩容。helpTransfer方法表示这个线程会帮着一起扩容。稍后再对扩容进行分析。

最后一个分支就是最长进入到的分支，即索引位上已经有一个元素了，需要把新的元素插入进去。

这里首先对f上了锁。这个f就是链表的第一个元素。在JDK7中，是对Segment元素进行上锁，其实是非常冗余的设计。在JDK8中，要修改哪个链表，就锁住哪个链表即可。JDK8对synchroned关键字本身也做了优化，因此可以这样用。

插入的操作也是对链表和红黑树做了区分，插入结束后，判断是否需要树化。流程上与JDK8的HashMap类似。不过，这里树化时的数据类型有所不同。这里的数据类型是TreeBin。

private final void treeifyBin(Node<K,V>[] tab, int index) {
    Node<K,V> b; int n, sc;
    if (tab != null) {
        if ((n = tab.length) < MIN_TREEIFY_CAPACITY)
            tryPresize(n << 1);
        else if ((b = tabAt(tab, index)) != null && b.hash >= 0) {
            synchronized (b) {
                if (tabAt(tab, index) == b) {
                    TreeNode<K,V> hd = null, tl = null;
                    for (Node<K,V> e = b; e != null; e = e.next) {
                        TreeNode<K,V> p =
                            new TreeNode<K,V>(e.hash, e.key, e.val,
                                              null, null);
                        if ((p.prev = tl) == null)
                            hd = p;
                        else
                            tl.next = p;
                        tl = p;
                    }
                    setTabAt(tab, index, new TreeBin<K,V>(hd));
                }
            }
        }
    }
}

因为树化也是修改操作，所以也是需要加锁的。加锁的逻辑在方法内部。可以看到树化的过程仍然是创建TreeNode，只不过最后把根结点封装到了TreeBin这个对象中。为什么要多封装一层呢？这是因为在树化过程中，根结点可能发生变化，所以封装一层后，外层的TreeBin就不会发生变化，保证整个操作中锁住的是同一个对象，而TreeBin内部的TreeNode根结点如何变化都没有影响。

private final void addCount(long x, int check) {
    CounterCell[] as; long b, s;
    if ((as = counterCells) != null ||
        !U.compareAndSwapLong(this, BASECOUNT, b = baseCount, s = b + x)) {
        CounterCell a; long v; int m;
        boolean uncontended = true;
        if (as == null || (m = as.length - 1) < 0 ||
            (a = as[ThreadLocalRandom.getProbe() & m]) == null ||
            !(uncontended =
              U.compareAndSwapLong(a, CELLVALUE, v = a.value, v + x))) {
            fullAddCount(x, uncontended);
            return;
        }
        if (check <= 1)
            return;
        s = sumCount();
    }
    if (check >= 0) {
        Node<K,V>[] tab, nt; int n, sc;
        while (s >= (long)(sc = sizeCtl) && (tab = table) != null &&
               (n = tab.length) < MAXIMUM_CAPACITY) {
            int rs = resizeStamp(n);
            if (sc < 0) {
                if ((sc >>> RESIZE_STAMP_SHIFT) != rs || sc == rs + 1 ||
                    sc == rs + MAX_RESIZERS || (nt = nextTable) == null ||
                    transferIndex <= 0)
                    break;
                if (U.compareAndSwapInt(this, SIZECTL, sc, sc + 1))
                    transfer(tab, nt);
            }
            else if (U.compareAndSwapInt(this, SIZECTL, sc,
                                         (rs << RESIZE_STAMP_SHIFT) + 2))
                transfer(tab, null);
            s = sumCount();
        }
    }
}

这里需要注意的是count的计数，不仅用到了baseCount字段，还用到了CounterCell数组。这也是效率上的考虑。过程大致是：如果CounterCell数组不为空或者对baseCount修改失败（多线程导致），则计算出索引位，对CounterCell数组中对应的值进行修改。也就是说baseCount和CounterCell数组中对应的值共同表示计数。从map的size方法可以看出：

public int size() {
    long n = sumCount();
    return ((n < 0L) ? 0 :
            (n > (long)Integer.MAX_VALUE) ? Integer.MAX_VALUE :
            (int)n);
}


final long sumCount() {
    CounterCell[] as = counterCells; CounterCell a;
    long sum = baseCount;
    if (as != null) {
        for (int i = 0; i < as.length; ++i) {
            if ((a = as[i]) != null)
                sum += a.value;
        }
    }
    return sum;
}

核心逻辑在transfer方法中，过程与HashMap基本一致。只是需要注意，在操作某一个Node时需要对TreeBin上锁。

由于只对TreeBin上锁，因此多线程同时扩容是可能发生的。之前见过的helpTransfer方法的含义就是当有一个线程正在扩容时，另一个帮助一起扩容。

final Node<K,V>[] helpTransfer(Node<K,V>[] tab, Node<K,V> f) {
    Node<K,V>[] nextTab; int sc;
    if (tab != null && (f instanceof ForwardingNode) &&
        (nextTab = ((ForwardingNode<K,V>)f).nextTable) != null) {
        int rs = resizeStamp(tab.length);
        while (nextTab == nextTable && table == tab &&
               (sc = sizeCtl) < 0) {
            if ((sc >>> RESIZE_STAMP_SHIFT) != rs || sc == rs + 1 ||
                sc == rs + MAX_RESIZERS || transferIndex <= 0)
                break;
            if (U.compareAndSwapInt(this, SIZECTL, sc, sc + 1)) {
                transfer(tab, nextTab);
                break;
            }
        }
        return nextTab;
    }
    return table;
}

核心逻辑也是transfer方法。
